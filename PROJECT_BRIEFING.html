Technical Project Briefing

A comprehensive guide for system handover and architecture overview of the Healthy Families Foundation Dashboard V2.

Project Mission

The Healthy Families Foundation (HFF) Dashboard is a specialized platform for tracking participant registration and attendance across health campaigns. It enables field workers to collect data in low-connectivity environments and sync it to a central repository.

Core Stack

React Vite Tailwind Dexie.js Supabase SQLite

Key Features

• Offline-first Registration
• Excel/Enketo XML Ingestion
• Real-time Demographics
• 18-Day Attendance Tracking

System Architecture

The system follows a hybrid architecture designed for resilience:

  • Frontend: React-based UI with Dexie.js for local data storage. This ensures field workers never lose data even during browser crashes or internet loss.
  • Data Layer: A complex sync manager orchestrates updates between local IndexedDB and Supabase.
  • Backend: A local Express server handles file processing and Google Sheets synchronization via a service account.

Normalization Engine

Automatically maps Setswana inputs (e.g., Monna/Mosadi) and numeric codes (1/2) to standard gender identifiers. It also features a "header scan" that finds the data table even if the Excel file has dynamic spacer rows.

Current Roadmap

Phase 2 of the project focuses on scaling and unification:

  1. Supabase Migration: Moving all local storage logic to a unified Postgres instance.
  2. PWA Support: Turning the dashboard into an installable mobile application.
  3. Responsive UI: Optimizing the attendance grid for tablet-based field supervision.
